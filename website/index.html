What it is

A better way to keep the web searchable.

FieldTheory for Windows moves the links you save out of the browser and into a local archive for research, references, and the things worth finding again later.

Capture Pull bookmarks from your browser profile into a local archive.
Find Surface old sources, half-remembered links, and research trails fast.
Own Keep the archive on your machine, under your control, on your terms.

Collect

Bring bookmarks in from the browser you already use.

Chrome and Brave are handled through the current Windows sync path so the archive stays private and portable.

Search

Find what matters fast, from the terminal.

SQLite FTS keeps the archive responsive, with command-line output that fits naturally into shell workflows.

Trust

Be clear about what ships.

The page points directly to the repo and install path, with no account layer, no lock-in, and no hidden backend.

How it works

Three moves from install to first search.

01

Install the package.

Use npm to install the current release globally on your Windows machine.

02

Index your browser.

Run the sync workflow to build a local archive from Chrome or Brave, with the data stored on your machine.

03

Search or script against it.

Use the CLI from the terminal, or hand the archive off to the rest of your stack when you need to go deeper.

FAQ

Short answers for release confidence.

What does FieldTheory actually do?

It turns browser bookmarks into a local archive you can query quickly from the command line.

Where does my data live?

On your machine. The workflow is local-first and avoids a hosted backend or account layer.

Is this a web app?

No. It’s a CLI with a landing page — the site is for the product, the repo, and the install path.
