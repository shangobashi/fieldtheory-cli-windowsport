FieldTheory for Windows

A local-first bookmark archive for the Windows terminal.

Import bookmarks from Chrome or Brave, store them in SQLite with full-text search, and query everything from the CLI. No hosted account. No sync service. Just your archive, on your machine.

Why

Bookmarks are usually saved. They are rarely retrievable.

FieldTheory turns browser bookmarks into a searchable local archive. Instead of leaving links scattered across folders and browser profiles, it brings them into a command-line workflow designed for Windows.

The point is not another dashboard. The point is getting old links back fast, with the archive still under your control.

Flow

Simple by design.

01

Import

Pull bookmarks from Chrome or Brave into one local archive.

02

Index

Store them in SQLite and prepare them for fast full-text search.

03

Search

Find saved links from the Windows terminal when you actually need them.

Usage

What FieldTheory does.

Browser import

Bring in existing bookmarks from Chrome and Brave without rebuilding anything by hand.

Local SQLite storage

Your archive stays in a format that is durable, inspectable, and portable.

Full-text terminal search

Query bookmarks quickly from the CLI using SQLite FTS.

No account layer

No hosted dashboard, no login wall, and no dependency on a remote sync surface.

Local means local

FieldTheory is not a hosted bookmarking service. It runs locally, stores data in SQLite, and keeps the workflow in your terminal.

Install

Archive your bookmarks. Search them locally.

Install the package, run the doctor command, sync the archive, and start searching from the terminal.

npm install -g fieldtheory-cli-windowsport
Then run: ftx doctor ftx sync ftx search "design systems"
