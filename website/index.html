FieldTheoryX for Windows

A local-first X/Twitter bookmark archive for the Windows terminal.

Use your logged-in Chrome or Brave session to sync bookmarks from X/Twitter, store them in SQLite with full-text search, and query everything from the CLI. No hosted layer. No remote index. Just your archive, on your machine.

Why

X bookmarks are easy to save. They are hard to retrieve.

FieldTheoryX turns X/Twitter bookmarks into a searchable local archive. Instead of leaving saved posts buried behind one scrolling interface, it brings them into a command-line workflow designed for Windows.

The point is not another dashboard. The point is getting saved links, threads, and references back fast, with the archive still under your control.

Simple by design.

Usage

What FieldTheoryX does.

X/Twitter bookmark sync

Sync your saved X/Twitter bookmarks into a local archive with the ftx sync command.

Chrome / Brave session support

Use your logged-in browser session for authentication, including the Windows DevTools fallback path when needed.

Local SQLite storage + search

Store everything locally and query it quickly from the CLI using SQLite FTS.

Classification + exploration

List, inspect, and explore the archive once it has been synced locally. AI-assisted classification is available as an experimental feature while final live Codex/Claude validation completes.

Local means local

FieldTheoryX is not a hosted bookmarking service. It uses your local browser session to sync X/Twitter bookmarks, stores them in SQLite, and keeps the workflow in your terminal.

Install

Sync your X/Twitter bookmarks. Search them locally.

Install the package, run the doctor command, sync your bookmarks from X/Twitter, and start searching from the terminal.

npm install -g fieldtheoryx-cli-windowsport
Then run: ftx doctor ftx sync ftx search "design systems"
