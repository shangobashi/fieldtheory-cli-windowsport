What it is

A bookmark tool built like software, not a dashboard.

FieldTheory for Windows keeps the original local-first idea, but reframes it as something quieter and more durable: a personal archive layer for the web that feels native on Windows instead of taped on.

Ingest Pull bookmarks from your browser profile into a local archive.
Recall Find old sources, half-remembered references, and research trails fast.
Own Keep the archive on your machine, under your control, on your terms.

Collect

Bring bookmarks in from the browser you already use.

Chrome and Brave are handled through the current Windows sync path so the archive stays private and portable.

Search

Recall from the terminal, where the work already lives.

SQLite FTS keeps the archive responsive, with command-line output that fits naturally into agent and shell workflows.

Trust

Stay source-first and transparent.

The page points to the repo and install path so the release story stays honest: no account, no lock-in, no hidden backend.

How it works

Three moves from install to archive.

01

Install the package.

Use npm to install the current release globally on your Windows machine.

02

Index your browser.

Run the sync workflow to build a local archive from Chrome or Brave, with the data stored on your machine.

03

Search or script against it.

Use the CLI from the terminal, or hand the archive to a broader agent workflow once the memory layer is live.

FAQ

Short answers for release confidence.

What does FieldTheory actually do?

It turns browser bookmarks into a local archive you can query quickly from the command line.

Where does my data live?

On your machine. The workflow is local-first and avoids a hosted backend or account layer.

Is this still a CLI?

Yes. The landing page is a launch surface for the CLI and the source, not a web app pretending to be one.
